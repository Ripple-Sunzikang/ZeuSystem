# 变更提案: IDE 重构与 CLI 全流程集成

## 需求背景
现有 IDE 功能残缺，无法覆盖从 C 编写到 bitstream/烧录的完整流程，且 UI 与交互体验不足以支撑频繁迭代。

## 产品分析

### 目标用户与场景
- **用户群体:** 课程设计学生与项目维护者
- **使用场景:** 高频编写 C 程序并快速编译、综合、烧录
- **核心痛点:** 操作分散、步骤不连贯、缺少统一可视化入口

### 价值主张与成功指标
- **价值主张:** 一体化 IDE 提供端到端流程入口与可视化反馈
- **成功指标:** IDE 内完成编译、汇编、bitstream 与烧录；错误反馈清晰

### 人文关怀
降低工具门槛，提供清晰的状态提示、日志输出与失败诊断。

## 变更内容
1. 重构 Electron IDE 前后端架构，引入现代编辑器组件
2. 集成 `seu` CLI 全流程操作（build/asm/bitstream/program/all）
3. 提升 UI 结构与任务编排，提供 VS Code 风格体验

## 影响范围
- **模块:** IDE、CLI、构建脚本、文档
- **文件:** `SEU-RISCV-CPU-IDE/*` 及相关文档
- **API:** IDE 与 CLI 交互 API
- **数据:** 无

## 核心场景

### 需求: IDE 一体化工作流
**模块:** IDE
在 IDE 内完成从 `.c` 到烧录的全流程。

#### 场景: C 编写与编译
用户在编辑器中编写 `.c` 文件，点击编译生成 `.s`。
- 预期结果: 生成 `.s` 并显示日志

#### 场景: 汇编生成 COE
用户点击汇编生成 `.coe` 并同步到 `rvTest/program.coe`。
- 预期结果: COE 生成并覆盖 Vivado 工程文件

#### 场景: 综合与实现
用户点击综合/实现/bitstream，生成 `.bit` 文件。
- 预期结果: bitstream 生成成功并可追踪状态

#### 场景: 烧录设备
用户点击烧录按钮完成 FPGA 编程。
- 预期结果: 设备烧录成功并返回状态

## 风险评估
- **风险:** GUI 调用 CLI 出错或阻塞
- **缓解:** 统一任务队列与日志输出，明确失败提示
